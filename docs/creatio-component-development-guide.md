# Building Custom Angular Components for Creatio Freedom UI

A step-by-step guide for building Angular components that integrate with Creatio's Freedom UI Designer, based on production experience with the forecast-hierarchy component.

## Architecture

The component is built as a **single Angular 17 project** using webpack Module Federation and `@creatio-devkit/common`. It:
- Appears in the **Freedom UI Designer** (drag-and-drop onto pages)
- Renders natively inside Creatio pages
- Receives data from Creatio datasources via page schema handlers
- Uses `ChangeDetectionStrategy.OnPush` to avoid Zone.js conflicts with Creatio's shell

**No AMD layer, no wrapper, no separate Angular versions.** Everything lives in one Angular 17 project generated by `clio new-ui-project`.

## Step 1: Deploy Creatio Locally

### 1.1 Start infrastructure containers

```bash
# PostgreSQL 17 on port 5436
DOCKER_CONFIG=/tmp podman run -d --name postgres-creatio \
  -p 5436:5432 -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=postgres \
  docker.io/postgres:17-alpine

# Valkey (Redis-compatible) on port 6379
DOCKER_CONFIG=/tmp podman run -d --name valkey-creatio \
  -p 6379:6379 docker.io/valkey/valkey:8-alpine
```

> Use `DOCKER_CONFIG=/tmp` to bypass credential store issues with Podman.

### 1.2 Configure clio

Run `clio open-settings` and add:

```json
{
  "db": {
    "local-postgres": {
      "DbType": "postgres",
      "Hostname": "localhost",
      "Port": 5436,
      "Username": "postgres",
      "Password": "postgres",
      "PgToolsPath": "/opt/homebrew/opt/libpq/bin",
      "Enabled": true
    }
  },
  "defaultRedis": "local-valkey",
  "redis": {
    "local-valkey": {
      "Hostname": "localhost",
      "Port": 6379,
      "Enabled": true
    }
  }
}
```

### 1.3 Deploy Creatio

```bash
clio deploy-creatio -e "creatio-local" --db pg --SitePort 9090 \
  --SiteName "creatio-local" --db-server-name local-postgres \
  --drop-if-exists --silent \
  --ZipFile "/path/to/Creatio_PostgreSQL.zip"
```

### 1.4 Start Creatio (if not running)

```bash
# Creatio 8.3.3 needs .NET 8 runtime explicitly:
cd <creatio-app-directory>
DOTNET_ROOT="/opt/homebrew/opt/dotnet@8/libexec" \
  /opt/homebrew/opt/dotnet@8/libexec/dotnet Terrasoft.WebHost.dll
```

Login at http://localhost:9090 with `Supervisor` / `Supervisor`.

### 1.5 Enable development mode

```bash
clio turn-fsm on -e creatio-local    # File system mode
clio install-gate -e creatio-local   # Clio API gateway
```

## Step 2: Create the Component Project

### 2.1 Create a Creatio package (in Creatio UI)

Go to Configuration > create a new package (e.g., `UsrMyComponentPackage`). Set Maintainer to "Customer". Add dependency on `CrtBase`.

### 2.2 Generate the Angular project

```bash
clio new-ui-project my_component -v usr \
  --package UsrMyComponentPackage --empty true \
  -e creatio-local --silent
```

This creates `projects/my_component/` with:
- `webpack.config.js` — Module Federation config
- `package.json` — Angular 17 + `@creatio-devkit/common`
- `src/app/app.module.ts` — `@CrtModule` + `bootstrapCrtModule`
- `angular.json` — build output goes to `packages/UsrMyComponentPackage/Files/src/js/my_component/`

### 2.3 Install dependencies

```bash
cd projects/my_component
npm install
npm install raw-loader --save-dev
```

### 2.4 Raise style budget

In `angular.json`, find `anyComponentStyle` budget and change:
```json
{ "type": "anyComponentStyle", "maximumWarning": "20kb", "maximumError": "30kb" }
```

## Step 3: Build the Component

### 3.1 Create the component

Create `src/app/view-elements/my-component/my-component.component.ts`:

```typescript
import {
  Component, ViewEncapsulation, ChangeDetectionStrategy, ChangeDetectorRef,
  Input, OnInit, OnChanges, OnDestroy, SimpleChanges, HostListener, ElementRef,
} from '@angular/core';
import { CrtViewElement, CrtInput, CrtInterfaceDesignerItem } from '@creatio-devkit/common';

@Component({
  selector: 'usr-my-component',
  encapsulation: ViewEncapsulation.ShadowDom,
  changeDetection: ChangeDetectionStrategy.OnPush,  // CRITICAL: prevents Zone.js loops
  template: `<!-- your template -->`,
  styles: [`
    @import url('https://fonts.googleapis.com/css2?family=Montserrat:wght@400;500;600&display=swap');
    :host { display: block; outline: none; }
    /* Font import MUST be inside the component styles — ShadowDom blocks external stylesheets */
    /* Use plain CSS selectors, NOT SCSS nesting — Angular 17 webpack doesn't support it in inline styles */
  `],
})
@CrtViewElement({
  selector: 'usr-my-component',
  type: 'usr.MyComponent',
})
@CrtInterfaceDesignerItem({
  toolbarConfig: {
    caption: 'My Component',
    name: 'usr_my_component',
    icon: require('!!raw-loader?{esModule:false}!./icon.svg'),
    defaultPropertyValues: {
      label: 'My Component',
    },
  },
})
export class MyComponentComponent implements OnInit, OnChanges, OnDestroy {
  @Input() @CrtInput() data: any[] = [];
  @Input() @CrtInput() someConfig: any = {};

  constructor(
    private hostEl: ElementRef<HTMLElement>,
    private cdr: ChangeDetectorRef,
  ) {
    this.hostEl.nativeElement.setAttribute('tabindex', '0');
  }

  ngOnInit(): void { /* initialize */ }

  ngOnChanges(changes: SimpleChanges): void {
    this.cdr.markForCheck();  // ALWAYS call this after input changes with OnPush
  }

  ngOnDestroy(): void { /* cleanup */ }

  // After any state mutation, always call:
  // this.cdr.markForCheck();
}
```

### 3.2 Create the icon

Create `src/app/view-elements/my-component/icon.svg`:
```svg
<svg width="20" height="20" viewBox="0 0 20 20" fill="none" xmlns="http://www.w3.org/2000/svg">
  <rect width="20" height="20" rx="2" fill="#FFD600"/>
</svg>
```

> Use a bright, distinctive color so you can find it in the designer.

### 3.3 Register in app.module.ts

```typescript
import { DoBootstrap, Injector, NgModule, ProviderToken, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
import { createCustomElement } from '@angular/elements';
import { BrowserModule } from '@angular/platform-browser';
import { bootstrapCrtModule, CrtModule } from '@creatio-devkit/common';
import { MyComponentComponent } from './view-elements/my-component/my-component.component';

@CrtModule({
  viewElements: [MyComponentComponent],
})
@NgModule({
  declarations: [MyComponentComponent],
  imports: [BrowserModule],
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
})
export class AppModule implements DoBootstrap {
  constructor(private _injector: Injector) {}

  ngDoBootstrap(): void {
    const element = createCustomElement(MyComponentComponent, { injector: this._injector });
    if (!customElements.get('usr-my-component')) {
      customElements.define('usr-my-component', element);
    }
    try {
      bootstrapCrtModule('my_component', AppModule, {
        resolveDependency: (token) => this._injector.get(<ProviderToken<unknown>>token),
      });
    } catch (e) { /* Not in Creatio context (local dev) */ }
  }
}
```

### 3.4 Build

```bash
npm run build
```

Output goes to `packages/UsrMyComponentPackage/Files/src/js/my_component/`.

## Step 4: Create the Bootstrap

### 4.1 bootstrap.js

Create `packages/UsrMyComponentPackage/Files/src/js/bootstrap.js`:

```javascript
(function () {
    /* Load the Module Federation remote module */
    var remoteEntryUrl = Terrasoft.getFileContentUrl(
        "UsrMyComponentPackage",
        "src/js/my_component/remoteEntry.js"
    );
    var script = document.createElement("script");
    script.src = remoteEntryUrl;
    script.type = "text/javascript";
    script.onload = function () {
        if (window.my_component) {
            var shareScopes = (typeof __webpack_share_scopes__ !== "undefined")
                ? __webpack_share_scopes__.default : {};
            try { window.my_component.init(shareScopes); } catch(e) { /* already initialized */ }
            console.log("my_component remote module loaded");
        }
    };
    document.head.appendChild(script);
})();
```

**Critical rules for bootstrap.js:**
- The package name in `Terrasoft.getFileContentUrl()` must EXACTLY match the Creatio package name
- Only call `.init()` — do NOT call `.get("./RemoteEntry")` — let Creatio handle that when rendering the component
- Wrap `.init()` in try/catch for "already initialized" errors
- Do NOT use `__webpack_share_scopes__` without checking `typeof` first

### 4.2 Files/descriptor.json

Create `packages/UsrMyComponentPackage/Files/descriptor.json`:

```json
{
    "bootstraps": ["src/js/bootstrap.js"]
}
```

```bash

SRC="packages/UsrMyComponentPackage/Files/src/js/my_component"
```

## Step 6: Use the Component on a Page

### 6.1 Place via designer

Open a record → gear icon → Edit page. Find your component in the left panel (look for the icon). Drag it onto the page. Save.

### 6.2 Feed data via handler

Edit the page schema to add a handler that reads data from a datasource and passes it to the component:

```javascript
define("UsrMyApp_ListPage", /**SCHEMA_DEPS*/[]/**SCHEMA_DEPS*/, function() {
  return {
    // ... viewConfigDiff, viewModelConfigDiff ...

    // Add an independent datasource for the component:
    modelConfigDiff: [
      // ... existing config ...
    ],

    // Add a collection attribute with embedded model:
    viewModelConfigDiff: [
      // ... existing attributes ...
      {
        "operation": "merge",
        "path": ["attributes"],
        "values": {
          "ComponentItems": {
            "isCollection": true,
            "modelConfig": { "path": "ComponentDS" },
            "viewModelConfig": {
              "attributes": {
                "CDS_Id": { "modelConfig": { "path": "ComponentDS.Id" } },
                "CDS_Name": { "modelConfig": { "path": "ComponentDS.UsrName" } },
                // ... other fields
              }
            },
            "embeddedModel": {
              "name": "ComponentDS",
              "config": {
                "type": "crt.EntityDataSource",
                "config": { "entitySchemaName": "UsrMyEntity" }
              }
            }
          }
        }
      }
    ],

    handlers: [
      // Feed data when collection loads
      {
        request: "crt.HandleViewModelAttributeChangeRequest",
        handler: async (request, next) => {
          const result = await next?.handle(request);
          if (request.attributeName !== "ComponentItems") return result;
          try {
            window._myComponentFeed?.(await request.$context.ComponentItems);
          } catch (e) { console.error("component:", e); }
          return result;
        }
      },
      // Set up the feed function + handle page re-entry
      {
        request: "crt.HandleViewModelInitRequest",
        handler: async (request, next) => {
          const unwrap = (v) => (v != null && typeof v === "object"
            && "__zone_symbol__value" in v) ? v.__zone_symbol__value : v;

          window._myComponentFeed = (items) => {
            if (!items || !items.length) return;
            const records = [];
            for (let i = 0; i < items.length; i++) {
              const a = items[i]?.attributes || {};
              records.push({
                id: String(unwrap(a.CDS_Id) || i),
                name: String(unwrap(a.CDS_Name) || ""),
                // ... map other fields
              });
            }
            const el = document.querySelector("usr-my-component");
            if (!el) return;
            el.data = records;
            // el.someConfig = { ... };
          };

          const result = await next?.handle(request);

          // Re-feed on page re-entry (e.g., returning from edit page)
          setTimeout(async () => {
            try {
              window._myComponentFeed?.(await request.$context.ComponentItems);
            } catch (e) { /* will be fed by attribute change handler */ }
          }, 500);
          return result;
        }
      },
      // Re-feed on refresh button click
      {
        request: "crt.LoadDataRequest",
        handler: async (request, next) => {
          const result = await next?.handle(request);
          setTimeout(async () => {
            try {
              window._myComponentFeed?.(await request.$context.ComponentItems);
            } catch (e) { /* ignore */ }
          }, 300);
          return result;
        }
      }
    ],
  };
});
```

**Key patterns:**
- Use `embeddedModel` for the datasource — this creates an independent instance, not shared with the list
- Use `HandleViewModelAttributeChangeRequest` to react to data load (fires when Items populate)
- Use `HandleViewModelInitRequest` to set up the feed function and handle page re-entry
- Use `crt.LoadDataRequest` to refresh data when user clicks the refresh button
- Unwrap Zone.js proxies: `__zone_symbol__value` contains the actual value
- Read from `items[i].attributes.CDS_FieldName`, not `items[i].CDS_FieldName`

## Critical Rules (Do NOT Violate)

### Component rules
1. **Always use `ChangeDetectionStrategy.OnPush`** — prevents Zone.js infinite loops with Creatio's shell
2. **Always call `this.cdr.markForCheck()`** after any state mutation
3. **Use `ViewEncapsulation.ShadowDom`** — required by Creatio for style isolation
4. **Import fonts inside component styles** — ShadowDom blocks external stylesheets
5. **Use plain CSS, not SCSS nesting** in inline styles — Angular 17 webpack doesn't expand nested selectors in inline component styles
6. **Default inputs to empty** (`[]`, `{}`, `''`) — don't include sample/demo data in defaults
7. **Guard `customElements.define()`** with `if (!customElements.get('tag-name'))` — prevents "already defined" errors

### Bootstrap rules
8. **Package name must match exactly** — `Terrasoft.getFileContentUrl("UsrMyComponentPackage", ...)` must use the exact Creatio package name
9. **Only call `.init()` in bootstrap** — do NOT call `.get("./RemoteEntry")` followed by `factory()` — this causes duplicate registration errors
10. **Wrap `.init()` in try/catch** — it throws if called twice
11. **Check `typeof __webpack_share_scopes__`** before using — not available on all pages

### Deployment rules
12. **Always use `clio push-pkg`** to deploy — it updates file content hashes and bootstraps. Manual file copies alone won't work because the hashes in `_FileContentDescriptors.js` won't match.
13. **Copy to BOTH `Pkg/` AND `conf/content/`** directories — `push-pkg` handles `Pkg/` but you may need to copy to `conf/content/` manually for the current session
14. **Creatio doesn't follow symlinks** — always use actual file copies

### Data handling rules
15. **Use `embeddedModel`** for component datasources — shared datasources cause cross-component interference
16. **Unwrap Zone.js proxies** — Creatio wraps values in `{__zone_symbol__state, __zone_symbol__value}` objects
17. **Read from `item.attributes.FIELD_NAME`** — not directly from `item.FIELD_NAME` (which returns a proxy)
18. **Use `setTimeout(..., 500)`** in `HandleViewModelInitRequest` for re-entry — data may not be ready immediately when returning from edit page

## Development Workflow

### Build and deploy cycle

```bash
# 1. Make changes to component
# 2. Build
cd projects/my_component && npm run build && cd ../..

# 3. Deploy
SRC="packages/UsrMyComponentPackage/Files/src/js/my_component"
PKG="<creatio-app>/Terrasoft.Configuration/Pkg/UsrMyComponentPackage/Files/src/js/my_component"
CONTENT="<creatio-app>/conf/content/UsrMyComponentPackage/src/js/my_component"
rm -rf "$PKG" "$CONTENT"
cp -r "$SRC" "$PKG" && cp -r "$SRC" "$CONTENT"

# 4. Push (updates hashes)
clio push-pkg packages/UsrMyComponentPackage -e creatio-local

# 5. Hard-refresh (Cmd+Shift+R) in browser
```

### Schema changes (handlers, viewConfigDiff)

Schema changes in `Pkg/*/Schemas/` are picked up on page refresh (FSM is on). No build or push needed.

After making changes in the Creatio UI designer, sync to filesystem:
```bash
clio pkg-to-file-system -e creatio-local
```

### Local testing

Add a test component and page to `src/app/test/` with the same sandbox layout (component on top, JSON config/data editors on bottom). In `app.module.ts`, detect `<app-test>` element for local dev mode:

```typescript
ngDoBootstrap(appRef: ApplicationRef): void {
  // Register custom element
  const element = createCustomElement(MyComponent, { injector: this._injector });
  if (!customElements.get('usr-my-component')) {
    customElements.define('usr-my-component', element);
  }

  // Local dev: render test page
  const testRoot = document.querySelector('app-test');
  if (testRoot) {
    appRef.bootstrap(TestComponent, testRoot);
    return;
  }

  // Creatio: register remote module
  try {
    bootstrapCrtModule('my_component', AppModule, {
      resolveDependency: (token) => this._injector.get(<ProviderToken<unknown>>token),
    });
  } catch (e) { }
}
```

Run `ng serve --port 4200` for local testing at http://localhost:4200.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Browser tab freezes | Missing `OnPush` change detection | Add `changeDetection: ChangeDetectionStrategy.OnPush` |
| Component not in designer | Bootstrap doesn't load `remoteEntry.js` | Check `bootstrap.js` creates `<script>` tag for `remoteEntry.js` |
| "Element with this type has already exist" | Duplicate `@CrtViewElement` registration | Don't call `factory()` in bootstrap — only `.init()` |
| "Mismatched anonymous define()" | JS wrapped in named `define()` | Use IIFE wrapper `(function(){...})()` |
| Styles don't apply | Font import in global styles | Move `@import url(...)` inside component `styles:[]` |
| 404 for component JS | Wrong package name in `Terrasoft.getFileContentUrl()` | Must match exact Creatio package name |
| Data not showing | Reading from proxy objects | Use `item.attributes.FIELD_NAME` and unwrap `__zone_symbol__value` |
| Data disappears after edit page | `HandleViewModelAttributeChangeRequest` doesn't re-fire | Add `HandleViewModelInitRequest` with `setTimeout` fallback |
| List and component share data | Same datasource instance | Use `embeddedModel` for independent datasource |
| `pg_restore` version error | Old PostgreSQL client | `brew install libpq`, set `PgToolsPath` |
| Creatio won't start | Wrong .NET version | Use `DOTNET_ROOT="/opt/homebrew/opt/dotnet@8/libexec"` |
